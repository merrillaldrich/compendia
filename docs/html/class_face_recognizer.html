Detailed Description

Detects and recognises faces across images using dlib's HOG detector and ResNet embeddings.

Converts QImages to the dlib pixel format, runs the HOG-based frontal face detector, aligns detected faces via a 5-point shape predictor, and computes 128-d embeddings using a pre-trained ResNet model. Results can be cached to disk to avoid redundant computation on repeated scans.

Parameters

Returns
A vector of (normalised QRectF, 128-d embedding) pairs, one per detected face.

The image is downscaled to at most kMaxDetectionDim on its longest edge before detection so the HOG sliding-window scan is fast even on large camera images. Because the scale is uniform, normalised coordinates derived from the downscaled dimensions are identical to those that would be obtained from the original, so no remapping is required.

◆ runSweep()

void FaceRecognizer::runSweep ( const QVector< TaggedFile * > & allFiles,
const QVector< TaggedFile * > & targetFiles,
std::function< Tag *(const QString &, const QString &)> tagFactory,
std::function< bool()> shouldCancel,
std::function< void()> processEvents )

Runs Phases 1–3 of the face recognition sweep synchronously on the calling thread.

Phase 1 seeds known-person embeddings from user-labeled regions across allFiles. Phase 2 clears all existing auto-detected face tags from targetFiles. Phase 3 scans every image in targetFiles, matches detected faces against known embeddings using matchThreshold_, and calls tagFactory to create new tags for unrecognised faces.

Parameters
allFilesAll files in the model (used to seed known embeddings).
targetFilesThe files to actually scan and tag.
tagFactoryFactory called as tagFactory(familyName, tagName) to create tags.
shouldCancelReturns true when the operation should abort.
processEventsCalled periodically to keep the UI responsive.

◆ scheduleEmbeddingWarmup()

void FaceRecognizer::scheduleEmbeddingWarmup ( TaggedFile * tf)

Triggers background known-face embedding warmup for tf if models are loaded.

Collects all user-labeled tag regions from tf, checks the known-face cache for missing embeddings, and launches a QtConcurrent task to compute them. Emits warmupScheduled(misses) just before the task starts so the caller can display a progress bar. Does nothing when a warmup is already running.

Parameters
tfThe file being deselected. May be nullptr.

◆ warmupKnownFaceEmbeddings()

void FaceRecognizer::warmupKnownFaceEmbeddings ( const QString & imagePath,
const QVector< std::tuple< QString, QString, QRectF > > & tagRegions )

Checks the known-face cache for each region in tagRegions and computes any missing embeddings, saving the updated cache to disk when done.

Intended to be called from a background thread (e.g. via QtConcurrent::run). Uses an internal mutex to prevent concurrent dlib inference from multiple threads. Emits embeddingWarmedUp() after each successfully computed (not cached) embedding.

Parameters
